# Product Guidelines: YouTube Stemmer

## Design Philosophy
- **User-Friendly & Intuitive:** The application should prioritize a clean, accessible interface that leverages familiar UI patterns and icons. Musicians should feel at home and be able to navigate the app's core functions with minimal learning curve.
- **Functional Simplicity:** While the underlying technology is complex, the user interface should remain focused on the user's immediate goal: retrieving and splitting songs for practice.

## User Experience (UX) Principles
- **Polished & Consistent UX:** Deliver a high-quality, professional experience that is consistent across all supported platforms (Desktop and Mobile). This includes smooth transitions, responsive design, and a unified visual language.
- **Immediate Feedback:** Provide clear visual cues for all user actions, especially during the audio retrieval and AI-powered stemming processes, ensuring the user is always aware of the system's state.
- **Dynamic Personalization:** Enhance the user experience with smart, adaptive features such as system-following themes and intelligent history suggestions that anticipate user needs.
- **Adaptive Responsiveness:** Ensure the application remains coherent and usable across varying screen sizes by enforcing sensible layout constraints and structural integrity.

## Tone and Voice
- **Informative & Helpful:** The application's interface and communication should be clear, informative, and helpful. Use plain language to explain features and guide the user through their workflow.
- **Supportive & Clear:** Maintain a supportive tone that empowers the musician to focus on their practice, rather than the technical details of the software.

## Error Handling and Resilience
- **Graceful & User-Friendly Errors:** When issues occur (e.g., network connectivity problems, AI processing errors), provide simple, solution-oriented messages that help the user resolve the situation or suggest a clear next step.
- **Stability and Recovery:** Prioritize application stability and ensure a smooth recovery process if an operation fails, minimizing any disruption to the user's practice session.
